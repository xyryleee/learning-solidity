# Exercise: Voting

Build a simple **on-chain voting** contract on Avalanche C-Chain.

## Requirements

- Proposals: store a list of proposal names or IDs.
- Voters: track who has voted (one vote per address per proposal).
- Vote: allow voting for a proposal by ID (e.g. support yes/no or just count votes).
- Result: function to get the vote count for a proposal (or winning proposal).

## What to do

1. Implement the contract in `Voting.sol` using the TODO hints.
2. Deploy to Fuji testnet and note the contract address.
3. In your PR, share your contract address and Snowtrace link.
